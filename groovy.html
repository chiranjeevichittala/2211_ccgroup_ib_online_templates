Write a Groovy script to demonstrate Groovy's support for multithreading.

Answer:

Multithreading in Groovy

  • Groovy supports multithreading and concurrent programming.
  • The 'Thread' class allows creating and managing threads in Groovy.
  • In the example, a closure 'myClosure' is defined to be executed in separate threads.
  • The closure prints messages and sleeps for 500 milliseconds in each iteration.
  • Two threads, 'thread1' and 'thread2', are created and started to execute the closure concurrently.
  • The 'join()' method is used to wait for both threads to finish before the script ends.
  • The output will show interleaved messages from both threads, demonstrating concurrent execution.

Code Example:

// Define a closure to execute in a separate thread
def myClosure = {
    for (int i = 1; i <= 5; i++) {
        println("Thread \${Thread.currentThread().name}: Iteration \$i")
        Thread.sleep(500)
    }
}

// Create and start multiple threads
def thread1 = new Thread(myClosure)
def thread2 = new Thread(myClosure)
thread1.start()
thread2.start()

// Wait for both threads to finish
thread1.join()
thread2.join()

This Groovy script demonstrates multithreading using the 'Thread' class and closures. A closure 'myClosure' is defined to print messages and sleep for 500 milliseconds in each iteration. Two threads, 'thread1' and 'thread2', are created and started to execute the closure concurrently. The 'join()' method is used to wait for both threads to finish before the script ends. The output will show interleaved messages from both threads, demonstrating concurrent execution.
