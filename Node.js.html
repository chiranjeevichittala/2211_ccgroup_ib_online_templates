Demonstrate how to implement server-sent events in an Express.js application.

Answer:

Implementing Server-Sent Events (SSE) in an Express.js Application

  • Set up an Express application and a route that will handle SSE connections.
  • Use the `res.writeHead()` method to set the appropriate headers for SSE.
  • Write SSE data to the client using the `res.write()` method with the appropriate event and data fields.
  • Set up a recurring interval to send updates to the client.

Code Example:

const express = require('express');

const app = express();

app.get('/events', (req, res) => {
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');

  const intervalId = setInterval(() => {
    const eventData = `data: ${JSON.stringify({ message: 'New update' })}\n\n`;
    res.write(eventData);
  }, 1000);

  req.on('close', () => {
    clearInterval(intervalId);
  });
});

const port = 3000;
app.listen(port, () => {
  console.log(`Server is running on port ${port}`);
});

This code snippet demonstrates how to implement Server-Sent Events (SSE) in an Express.js application. The `/events` route is configured to send SSE data to connected clients every second. The headers are set to indicate that the response will be in SSE format. The interval function sends SSE data using the `res.write()` method. The `req.on('close')` event is used to clear the interval when the client connection is closed.

Real-time Usage:

Explain the difference between `readFile` and `createReadStream` in Node.js.

Answer:

Difference Between `readFile` and `createReadStream`

  • `readFile` is a method provided by the 'fs' module to read the entire contents of a file into memory.
  • It's suitable for reading small files where you want to process the entire content at once.
  • `readFile` can lead to performance issues if used for large files since it loads the entire content into memory.
  • `createReadStream` is a method provided by the 'fs' module to create a readable stream from a file.
  • It's suitable for reading large files because it reads and processes the file in chunks.
  • `createReadStream` is memory-efficient and non-blocking, making it ideal for processing big files.
  • It's especially useful for reading log files, streaming data, or processing data while it's being read.
  • Choosing between the two depends on the size of the file and the desired memory and performance characteristics.

Real-time Usage:

  • Using `createReadStream` to process a large log file in chunks without loading it all into memory.
  • Using `readFile` to read a small text file and process its content.
  • Using `createReadStream` to stream video or audio files for playback.

Question:

Explain how the single-threaded model in Node.js works.

Answer:

Single-Threaded Model in Node.js

  • Node.js applications run on a single main thread, which is used for executing JavaScript code.
  • Asynchronous operations are handled using the event loop, allowing the main thread to efficiently handle multiple I/O-bound tasks.
  • When a task is initiated, it is added to the event loop's queue.
  • The event loop continuously checks the queue for pending tasks, executing them when the main thread is available.
  • I/O operations, like file reading or network requests, do not block the main thread, enhancing application responsiveness.
  • Callbacks and event listeners are used to manage the flow of execution.
  • While Node.js is single-threaded, it can still leverage multi-core systems by creating child processes for parallelism.
  • Node.js excels in I/O-bound tasks, but CPU-bound tasks might benefit from other approaches.
  • The single-threaded model simplifies development but requires careful handling of CPU-intensive operations to prevent blocking the event loop.

Real-time Usage:

  • Building scalable web servers handling multiple concurrent connections.
  • Creating real-time applications like chat systems that need to handle numerous simultaneous events.
  • Optimizing I/O-bound operations, such as reading and writing files, without blocking the main thread.

Code Example:

const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

const users = [{
  id: 1,
  username: 'john',
  password: 'password'
}];

// Configure the LocalStrategy
passport.use(new LocalStrategy(
  (username, password, done) => {
    const user = users.find(u => u.username === username);
    if (!user || user.password !== password) {
      return done(null, false, { message: 'Incorrect username or password' });

  }
));

// Serialize and deserialize user
passport.serializeUser((user, done) => done(null, user.id));
passport.deserializeUser((id, done) => {
  const user = users.find(u => u.id === id);
  done(null, user);
});

// Authenticate using passport
app.post('/login',
  passport.authenticate('local', {
    successRedirect: '/dashboard',
    failureRedirect: '/login',
    failureFlash: true
  })
);

const port = 3000;
app.listen(port, () => {
  console.log(`Server is running on port ${port}`);
});

This code snippet demonstrates how to set up a basic passport authentication strategy using the `passport` and `passport-local` packages. The `LocalStrategy` is configured to authenticate users based on a predefined list of users. Answer:

Advised to Keep CPU-Bound Tasks Out of Node.js

  • Node.js runs in a single thread and is optimized for handling I/O-bound operations.
  • CPU-bound tasks are computationally intensive and can monopolize the event loop, blocking other I/O operations.
  • When a CPU-bound task runs in Node.js, it prevents other tasks from making progress.
  • Blocking the event loop hinders the responsiveness of the application and affects its scalability.
  • Node.js is not well-suited for parallel execution of CPU-bound tasks.
  • To utilize multicore CPUs, Node.js applications can spawn child processes for CPU-bound tasks.
  • CPU-bound tasks are better handled by languages that support multithreading or parallel execution.
  • In Node.js, asynchronous tasks are preferred, as they allow the event loop to continue processing I/O operations.

Real-time Usage:

  • Computationally intensive mathematical calculations.

  • Video encoding or image processing that utilizes the CPU heavily.

  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('Hello, World!');
  }).listen(8000);
  console.log(`Worker ${process.pid} started`);
}

  });

This code snippet demonstrates how to use Promises to read multiple files asynchronously and log their contents in order.
