Explain what Optional Chaining is in TypeScript and demonstrate how to use it.

Answer:

Optional Chaining in TypeScript

  • In traditional JavaScript, accessing a property or method of a deeply nested object that is null or undefined would result in a runtime error.
  • Optional Chaining allows you to use the '?' operator to safely access properties and methods without triggering an error if an intermediate value is null or undefined.
  • If the property or method exists in the chain, the result is the property value or method return value. If any intermediate value is null or undefined, the result is undefined.
  • The syntax for Optional Chaining is 'object?.property' or 'object?.method()'.
  • The '?.' operator can be used multiple times in a chain to handle deep nested properties and methods.
  • Optional Chaining is especially useful when dealing with data fetched from APIs or databases, where certain properties may be missing.
  • Optional Chaining works in conjunction with the 'nullish coalescing' operator (??) to provide default values for undefined or null properties.
  • The '??' operator allows you to specify a fallback value if the result of the Optional Chaining is null or undefined.
  • By combining Optional Chaining with the '??' operator, you can safely access deeply nested properties and provide default values in case of missing data.
  • TypeScript performs static analysis on Optional Chaining expressions, providing type checking and safety during development.
  • It's important to use Optional Chaining carefully and consider providing appropriate default values or handling null/undefined cases accordingly.
  • In conclusion, Optional Chaining simplifies and enhances the safety of working with potentially missing properties and methods in TypeScript.

Real-time Usage:

  • Using Optional Chaining to access nested properties of a user object (user?.address?.city).
  • Using Optional Chaining to call a method on a deeply nested object (object?.property?.method()).
  • Using Optional Chaining to handle API responses with varying data structures (response?.data?.results).
  • Using Optional Chaining to access properties of configuration objects (config?.settings?.theme).
  • Using Optional Chaining to navigate through complex navigation routes (location?.state?.params?.id).
  • Using Optional Chaining to access properties of user preferences (preferences?.language).
  • Using Optional Chaining to handle nested data in JSON objects (data?.property?.subproperty).
  • Using Optional Chaining to safely access properties in deeply nested arrays (array[0]?.nested?.property).
  • Using Optional Chaining to access properties of deeply nested settings objects (settings?.config?.options).
  • Using Optional Chaining to handle data from external sources that may have incomplete data (source?.data?.field).

  return new Person(...args);

In this code, we define a class 'Person' with a constructor that takes 'name' and 'age' as arguments. What is the difference between '==' and '===' operators in TypeScript?

Answer:

Difference between '==' and '===' operators in TypeScript

  • 1. Loose Equality (==): The '==' operator checks for equality after performing type coercion. It converts operands to the same type and then compares their values. It is called loose equality or abstract equality comparison.
  • 2. Strict Equality (===): The '===' operator checks for equality without performing type coercion. It requires both operands to have the same type and the same value. It is called strict equality or identity comparison.
  • 3. Preferred Usage: In most cases, it is recommended to use the '===' operator (strict equality) to avoid unexpected type coercion and ensure more predictable comparisons.

Code Example:

let value1: number | string = 5;
let value2: number | string = '5';

console.log(value1 == value2); // Output: true
console.log(value1 === value2); // Output: false

In this example, we have two variables 'value1' and 'value2', where 'value1' holds the number 5, and 'value2' holds the string '5'. The '==' operator (loose equality) compares the values of the operands after type coercion and returns true because 5 (number) is considered equal to '5' (string) after type conversion. On the other hand, the '===' operator (strict equality) checks for both the value and type, so it returns false since the operands have different types.

Real-time Usage:

  • Loose Equality: The '==' operator is used when you want to compare values but allow type coercion. For example, '5' == 5 evaluates to true.
  • Strict Equality: The '===' operator is used when you want to compare values without allowing type coercion. For example, '5' === 5 evaluates to false.
  • Avoiding Bugs: Using the '===' operator helps avoid bugs caused by unintentional type conversions.

Answer:

Using 'Mapped Types' in TypeScript

  • Mapped types are defined using the 'keyof' operator in combination with the 'in' keyword.
  • The 'keyof' operator retrieves a union type of all possible property names of the source type.
  • By using 'keyof' and 'in', you can apply a transformation to each property of the source type.
  • Common mapped types include 'Readonly', 'Partial', 'Pick', and 'Record'.
  • The 'Readonly' mapped type creates a new type with all properties set to 'readonly', ensuring immutability.
  • The 'Partial' mapped type creates a new type with all properties set to optional, allowing partial updates.
  • The 'Pick' mapped type creates a new type with only selected properties from the source type.
  • The 'Record' mapped type creates a new type with properties based on a union of property names and their types.
  • Mapped types are often used in utility functions and generic types to enhance type safety and code reusability.
  • Using mapped types provides a powerful way to define complex types and handle data transformations in a type-safe manner.

Code Example:

    name: string;
    age: number;
}

type PartialPerson = Partial<Person>

type NameOnly = Pick<Person, 'name'>

type RecordPerson = Record<'info' | 'data', Person>

const person: Person = {
    name: 'John Doe',
    age: 30
};

const partialPerson: PartialPerson = person;

const nameOnly: NameOnly = {

  • By changing the strategy at runtime, the context class can switch between different algorithms without modifying its code.
  • The Strategy Pattern promotes flexibility, extensibility, and promotes the open/closed principle.
  • It allows for easy addition of new algorithms or modification of existing ones.
  • In TypeScript, you can use interfaces to define the strategy contract and classes to implement specific strategies.
  • The Strategy Pattern is widely used in scenarios where there are multiple ways to accomplish a task, and the choice of the algorithm may vary at runtime.
  • It is one of the GoF design patterns, promoting a flexible and maintainable approach to handling algorithms.
