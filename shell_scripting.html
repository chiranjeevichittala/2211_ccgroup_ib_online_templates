Question:

Write a shell script to check if a given number is prime.

Answer:

Shell Script: Check Prime Number

  • 1. Open a text editor and create a new file with a '.sh' extension, like 'check_prime.sh'.
  • 2. Add a shebang line at the beginning to specify the shell to use, such as '#!/bin/sh'.
  • 3. Prompt the user to enter a number using the 'read' command: 'read -p "Enter a number: " num'.
  • 4. Use a loop to check if the number is divisible by any integer from 2 to the square root of the number.
  • 5. If the number is divisible by any integer, it is not prime; otherwise, it is prime.
  • 6. Print the result to the console using 'echo'.
  • 7. Save the file and exit the text editor.
  • 8. Make the script executable using the 'chmod' command: 'chmod +x check_prime.sh'.
  • 9. Execute the script by running: './check_prime.sh'.

Code Example:

#!/bin/sh

read -p "Enter a number: " num

is_prime=true
for ((i=2; i*i <= num; i++))
  do
    if ((num % i == 0))
    then
      is_prime=false
      break
    fi
  done

if $is_prime
then
  echo "$num is a prime number."
else
  echo "$num is not a prime number."
fi

The script starts with the shebang line '#!/bin/sh' to specify the shell interpreter as 'sh'. The 'read' command is used to prompt the user to enter a number, which is stored in the 'num' variable. The script then uses a 'for' loop to iterate from 2 to the square root of the number. If the number is divisible by any integer within this range, the 'is_prime' variable is set to false. Finally, the script uses the 'echo' command to print the result based on the value of 'is_prime'.

Real-time Usage:

  • Validating user input to ensure it is a prime number.
  • Checking the primality of a number in an algorithm or program.
  • Generating a list of prime numbers within a specific range.

Answer:

Passing Arguments to a Shell Script

  • 1. Command-Line Syntax: When executing the shell script, provide the arguments after the script name, separated by spaces. For example, 'myscript.sh arg1 arg2 arg3'.
  • 2. Accessing Arguments: Inside the script, you can access the passed arguments using special variables. The most commonly used variables for this purpose are '$1', '$2', '$3', and so on, where '$1' represents the first argument, '$2' represents the second argument, and so on.
  • 3. \$@ and \$*: Additionally, you can use '\$@' or '\$*' to represent all the passed arguments as a single string. For example, 'for arg in "\$@"' will iterate over all the arguments.
  • 4. Number of Arguments: The total number of arguments passed to the script can be obtained using '\$#'.
  • 5. Example: For instance, if you run the script as 'myscript.sh hello world', '\$1' will be 'hello', '\$2' will be 'world', and '\$#' will be '2'.
  • 6. Quoting Arguments: When passing arguments that contain spaces or special characters, it's essential to enclose them in quotes. For example, 'myscript.sh "arg with spaces"'.
  • 7. Shift Command: The 'shift' command can be used to discard the first argument and make the next argument accessible as '\$1'. This is useful when processing arguments in a loop.
  • By passing arguments to shell scripts, you can customize the behavior and functionality of the script based on user input.

  • 5. Ignored by Non-Unix Systems: Non-Unix systems, such as Windows, typically ignore the shebang line, treating it as a comment. As a result, shell scripts are usually executed by explicitly specifying the interpreter.
  • 6. Optional but Recommended: While the shebang line is not mandatory for shell scripts, it is a recommended practice to include it. It ensures that the script runs with the intended shell, and it makes the script self-contained.

Write a shell script to find the largest file in a directory.

Answer:

Shell Script: Find Largest File in a Directory

  • 1. Open a text editor and create a new file with a '.sh' extension, like 'find_largest_file.sh'.
  • 2. Add a shebang line at the beginning to specify the shell to use, such as '#!/bin/sh'.

  • 4. Use the 'find' command to locate all the files in the directory and its subdirectories: 'find "$dir_path" -type f'.
  • 5. Pipe the output of 'find' to 'du' command to calculate the size of each file in kilobytes: 'find "$dir_path" -type f | xargs du -k'.
  • 6. Sort the results in reverse order to find the largest file: 'sort -nr'.
  • 7. Use 'head' command to display the first line of the sorted list, which will be the largest file: 'head -n 1'.

  • 9. Make the script executable using the 'chmod' command: 'chmod +x find_largest_file.sh'.

#!/bin/sh

read -p "Enter the directory path: " dir_path
largest_file=$(find "$dir_path" -type f | xargs du -k | sort -nr | head -n 1)
echo "The largest file in the directory is: $largest_file"

The script starts with the shebang line '#!/bin/sh' to specify the shell interpreter as 'sh'. The 'read' command is used to prompt the user to enter the directory path, which is stored in the 'dir_path' variable. The 'find' command is then used to locate all the files in the directory and its subdirectories. The output is piped to the 'du' command to calculate the size of each file in kilobytes. The results are then sorted in reverse order using 'sort -nr' to find the largest file. Finally, the 'head' command is used to display the first line of the sorted list, which will be the largest file, and it is stored in the 'largest_file' variable. The script prints the path of the largest file at the end.

Question:

Explain the concepts of subshells and child shells in shell scripting, and highlight their differences.

Answer:

Understanding Subshells and Child Shells in Shell Scripts

  • 1. Subshell: A subshell is a new instance of the shell that is created within the current shell process. It is used to execute commands or scripts in a new environment without affecting the parent shell's environment. Subshells are created when you use parentheses '( )' to group commands or when using certain constructs like 'command1 | command2', which runs 'command1' and 'command2' in separate subshells.
  • 2. Isolated Environment: Subshells have their own process IDs (PIDs), and any changes made to the environment within a subshell do not affect the parent shell or other subshells.
  • 3. Command Execution: When a script or command is executed in a subshell, it inherits a copy of the parent shell's environment. However, any changes made to variables or settings in the subshell do not propagate back to the parent shell.

#!/bin/sh

if [ -z "$ip_address" ]; then

fi

echo "Server IP address: $ip_address"

The script starts with the shebang line '#!/bin/sh' to specify the shell interpreter as 'sh'. The script attempts to find the IP address of the server using the 'ifconfig' command. The 'grep -Eo' command uses a regular expression to find IP addresses in the 'ifconfig' output. The 'awk' command selects the second column, which contains the IP address. The 'grep -v' command filters out the loopback address '127.0.0.1'. If the 'ifconfig' command is not available, the script uses the 'hostname' command with the '-I' option to get the IP address.
