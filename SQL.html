Question:

Explain the differences between `CHAR` and `VARCHAR` data types.

Answer:

Explain the differences between `CHAR` and `VARCHAR` data types.

  • `CHAR` is a fixed-length data type, meaning that it reserves space for characters based on the length specified, regardless of the actual size of the data.
  • `VARCHAR` stands for Variable Character. It is a variable-length data type, which means it reserves space for the actual data stored, plus some additional overhead for managing the length.
  • Using `CHAR` can be more space-efficient if most of the values are close to the same length.
  • Using `VARCHAR` is generally more space-efficient when storing strings with a large variance in lengths.

Real-time Usage:

  • For storing fixed data like country codes (e.g., 'US', 'UK'), `CHAR` can be more suitable.
  • For storing varying data like user comments or descriptions, `VARCHAR` would be a better choice.

Concept of Database Lock

  • Purpose of Locks: Ensure data consistency and integrity by preventing conflicting operations.

Real-time Usage:

  • Applying an exclusive lock on a 'Orders' table while updating an order's status to prevent concurrent updates.
  • Using shared locks when multiple transactions read data from a 'Products' table to ensure data consistency.

Describe the different normal forms in database design.

Answer:

Describe the different normal forms in database design.

  • First Normal Form (1NF): Ensures each column contains atomic, indivisible values and has a unique primary key.
  • Second Normal Form (2NF): All non-key attributes are fully functionally dependent on the primary key.
  • Third Normal Form (3NF): Every non-prime attribute is non-transitively dependent on every key of the table.
  • Boyce-Codd Normal Form (BCNF): A stricter version of 3NF.
  • Fourth Normal Form (4NF): Concerns itself with Multi-Valued Dependencies.
  • Fifth Normal Form (5NF) or Project-Join Normal Form (PJNF): Deals with cases where information can be reconstructed from smaller pieces.

Real-time Usage:

  • When designing a library management system, normalization will ensure that book details, borrower details, and borrowing records are stored in a manner that prevents data redundancy and inconsistencies.
  • In a school management database, teacher details, student details, and their respective courses would be normalized to prevent data anomalies and ensure data integrity.

Data Consistency in SQL
