Question:

Write a Perl script to check if a number is prime.

Answer:

Checking Prime Numbers in Perl

  • Define a function (e.g., 'is_prime') that takes the number to be checked as an argument.
  • Check if the input number is less than 2; if so, return '0' as numbers less than 2 are not prime.
  • Use a 'for' loop to iterate from 2 to the square root of the input number.
  • Within the loop, check if the input number is divisible by the loop variable ('i') using the modulo operator (%). If it is divisible, return '0' as the number is not prime.
  • If no divisors are found after the loop, return '1' to indicate that the number is prime.
  • Use 'print' function to display whether the number is prime or not, based on the function's return value.

Code Example:

sub is_prime {
    my ($num) = @_;
    if ($num < 2) {
        return 0;  # Numbers less than 2 are not prime
    }
    for (my $i = 2; $i * $i <= $num; $i++) {
        if ($num % $i == 0) {
            return 0;  # Found a divisor, not prime
        }
    }
    return 1;  # No divisors found, prime number
}

# Test cases
my $number1 = 17;
my $number2 = 20;

if (is_prime($number1)) {
    print "\$number1 is a prime number.\n";
} else {
    print "\$number1 is not a prime number.\n";
}

if (is_prime($number2)) {
    print "\$number2 is a prime number.\n";
} else {
    print "\$number2 is not a prime number.\n";
}

This script defines a function 'is_prime' that checks whether a given number is prime in Perl. The function first checks if the input number is less than 2; if so, it returns '0' as numbers less than 2 are not prime. Then, it uses a 'for' loop to iterate from 2 to the square root of the input number. Within the loop, it checks if the input number is divisible by the loop variable ('i') using the modulo operator (%). If it is divisible, the function returns '0', indicating that the number is not prime. If no divisors are found after the loop, the function returns '1', indicating that the number is prime. Test cases with 'number1 = 17' and 'number2 = 20' are provided to demonstrate the usage of the 'is_prime' function, and 'print' function is used to display whether the numbers are prime or not based on the function's return value.

How would you check if a string is a palindrome in Perl?

Answer:

Checking Palindrome in Perl

  • Define a function (e.g., 'is_palindrome') to perform the palindrome check.
  • Remove any non-alphanumeric characters from the input string using a regular expression (e.g., 's/[^a-zA-Z0-9]//g').
  • Convert the input string to lowercase using 'lc' to make the comparison case-insensitive.
  • Create a new string that is the reverse of the input string using 'reverse' function.
  • Compare the original input string with its reverse using the 'eq' operator.
  • Return 'true' (1) if the strings match, indicating that the input string is a palindrome, otherwise, return 'false' (empty string or undef).

Code Example:

sub is_palindrome {
    my ($input) = @_;
    $input =~ s/[^a-zA-Z0-9]//g;  # Remove non-alphanumeric characters
    $input = lc($input);  # Convert to lowercase
    my $reverse_input = reverse($input);
    return $input eq $reverse_input;
}

if (is_palindrome($str2)) {
    print "\$str2 is a palindrome.\n";
} else {
    print "\$str2 is not a palindrome.\n";
}

This script defines a function 'is_palindrome' that checks if a given string is a palindrome in Perl. The input string is passed as an argument to the function. First, any non-alphanumeric characters are removed from the input string using the regular expression 's/[^a-zA-Z0-9]//g'. Then, the input string is converted to lowercase using 'lc' function for a case-insensitive comparison. Next, the reverse of the input string is obtained using 'reverse' function, and the original input string is compared with its reverse using the 'eq' operator. The function returns 'true' (1) if the strings match, indicating that the input string is a palindrome, otherwise, it returns 'false'. Test cases are provided to demonstrate the function's usage with different input strings.
