Question:

Describe the differences between Amazon S3 and Amazon EBS.

Answer:

Amazon S3 vs. Amazon EBS

  • S3 is object storage, while EBS is block storage.
  • S3 is suitable for storing files, backups, and media, while EBS is used for attaching storage volumes to EC2 instances.
  • S3 is scalable and designed for high availability, while EBS volumes are attached to specific instances.
  • S3 is accessible via a web API, while EBS volumes are attached as block devices to EC2 instances.

Real-time Usage:

  • A company can use Amazon S3 to store and serve static assets for a website, such as images, videos, and stylesheets.
  • An enterprise can employ Amazon EBS to provide additional storage to their database servers hosted on EC2 instances.
Keywords: Amazon S3, Amazon EBS, Differences, AWS Learn more

Question:

What is the difference between an Elastic IP and a Public IP?

Answer:

Elastic IP vs. Public IP

  • Elastic IP: It is a static, persistent IP address that can be allocated to and reassigned between AWS instances. It helps ensure that an instance retains the same IP address even after stopping and starting.
  • Public IP: It is a dynamic IP address assigned to an AWS instance when it is launched. Public IPs may change when an instance is stopped and started.
  • Elastic IPs are primarily used when you need a fixed IP address for resources like web servers, while Public IPs are assigned by default.

Real-time Usage:

  • A web server using an Elastic IP maintains a consistent IP address, ensuring uninterrupted access.
  • For temporary testing or development instances, a Public IP may be sufficient due to its dynamic nature.
Keywords: Elastic IP, Public IP, Differences, AWS Learn more

Question:

What are AWS security groups?

Answer:

AWS Security Groups

  • Inbound Rules: Security groups allow you to define inbound rules to control incoming traffic based on source IP addresses and ports.
  • Outbound Rules: You can also configure outbound rules to control outbound traffic from instances.
  • Stateful: Security groups are stateful, meaning that if you allow inbound traffic from a specific IP, corresponding outbound traffic is automatically allowed.
  • Instance-Level: Security groups are associated with instances and apply at the instance level.

Real-time Usage:

  • An EC2 instance hosting a web application can have a security group that allows inbound traffic on port 80 (HTTP) and 443 (HTTPS) from all IP addresses.
  • Security groups can be used to enforce network segmentation and control traffic between different application tiers.
